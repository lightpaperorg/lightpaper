# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in lightpaper.org, please report it responsibly.

**Email:** security@buildworld.ai

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **48 hours**: Acknowledgment of your report
- **7 days**: Initial assessment and response
- **30 days**: Target for fix deployment (critical issues prioritized)

## Scope

The following are in scope:
- lightpaper.org web application and API
- Authentication and authorization bypasses
- SQL injection, XSS, CSRF, and other OWASP Top 10
- Data exposure or leakage
- Rate limiting bypasses

The following are out of scope:
- Social engineering attacks
- Denial of service (DoS/DDoS)
- Issues in third-party dependencies (report to upstream)
- Issues requiring physical access

## Recognition

We appreciate responsible disclosure. With your permission, we will acknowledge your contribution in our changelog.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (main branch) | Yes |
| Older releases | No |
